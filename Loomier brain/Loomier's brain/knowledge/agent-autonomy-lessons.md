# Agent Autonomy — Lessons Learned

תאריך: 2026-05-09
הקשר: בדיקה של סוכן ה-MCQ (Questions Agent) חשפה שהוא חזר על אותן 7 שאלות בלי קשר לסוגת המשחק.

---

## הבעיה שגילינו

שלחנו לסוכן שני פרומפטים שונים מאוד:
1. "משחק חלל שבו שחקן יורה באסטרואידים"
2. "משחק פאזל עם בלוקים צבעוניים שצריך להתאים שלושה בשורה"

**שתי התשובות חזרו עם אותן 7 קטגוריות זהות:**
מטרת שחקן · מימד/מצלמה · בקרה · core loop · נכסי אמנות · קושי · התאמה למובייל.

הסוכן לא חשב על המשחק — הוא מילא צ'קליסט.

---

## האשמה — לא הייתה במודל. הייתה בקוד שלנו.

`gpt-5-mini` בהחלט מסוגל לחשוב יצירתית. אנחנו אסרנו עליו.

### גורם #1 — User prompt עם רשימה מפורשת
`prototype/backend/src/services/promptService.js` (לפני):
```js
'Prioritize missing design decisions:
  player goal, camera/dimension, controls, core loop,
  art/assets, difficulty, and mobile fit.'
```
המודל קיבל את ה-7 הללו ככוכב הצפון. הוא ציית.

### גורם #2 — System prompt עם 3 דרישות-אגנוסטיות-לסוגה
- "Consider 2D, 3D, and hybrid... Phaser/Three/Rapier" → תמיד שאלת מימד
- "Consider existing vs AI-generated assets" → תמיד שאלת אסטים
- "Prefer mobile-first" → תמיד שאלת מובייל

3 שאלות **מובטחות** בכל ריצה, בלי קשר לרעיון.

### גורם #3 — Schema שאוסר חופש
`prototype/backend/src/schemas/apiSchemas.js`:
- `options.min(1).max(2)` — הכל בינארי A/B (אי אפשר 4 וריאציות אווירה)
- `questions.min(5).max(8)` — מאלץ לכסות 5-8 דומיינים גם ברעיון פשוט
- `label.max(300)` — ניואנס מצומצם

### גורם #4 — Temperature שמרני
`prototype/backend/src/routes/mcq.js`: `temperature: 0.45`
זה אזור "תשובות צפויות". יצירתיות אמיתית מתחילה ב-0.7+.

### גורם #5 — Mock fallback מחזק את התבנית
`prototype/backend/src/services/fallbackAIService.js`:
ה-fallback פולט 6 שאלות גנריות זהות. כשהוא נכנס לפעולה (rate-limit/credits), הוא בעצם **מאמן את המשתמש** לצפות לתבנית הזו.

## הוכחה — אחרי השינוי

פרומפט: *"Rhythm game: tap to drumbeats to fight sea monsters from a ship"*

הסוכן החזיר 6 שאלות, **כולן ייחודיות לרעיון**, **3 אופציות לכל אחת**:
1. מורכבות-קלט (טאפ-יחיד / 2-4 פדים / מערכת תופים מלאה)
2. מיפוי-תופים-ללחימה (התקפות / אפקטים-לפי-תוף / הגנה)
3. תפקיד-ריתמי-של-מפלצות (טלגרף-קצב / לא-ריתמי / נקודות-תורפה)
4. בעלות-על-המוזיקה (קומפוזיציה / אלתור / היברידי)
5. עיצוב-טמפו (קבוע / משתנה-לפי-שלב / פולירית' סביבתי)
6. שחקנים-ותפקידים (סולו / קו-אופ / תחרותי)

אפס שאלות מהתבנית הישנה. אפס שאלות על מובייל/מימד/אסטים.

## הלקח הרחב — מתי "הוראה מועילה" הופכת ל"כלא"

יש פיתוי לכתוב prompts כמו specs:
> "ודא שכיסית את A, B, C, D, E, F, G."

זה מרגיש בטוח — בודק לעצמך שום דבר חשוב לא ייפול. אבל **המודל יקבל את הרשימה כתוצר הסופי המבוקש**, לא כרשימת אזהרה.

החוק שהפנמנו:

> **אם אתה מספק למודל רשימה מפורשת, אל תצפה ליצירתיות.**
> **אם אתה רוצה יצירתיות, ספק קריטריון איכות + דוגמת-נגד + רשות לסטות.**

נוסחה שעובדת:
1. **מסגרת חיובית** — "תחשוב כ-X" (לא "תכסה את A-G")
2. **אנטי-תבנית מפורשת** — "אל תיפול לצ'קליסט. אם זה מרגיש כמו הצ'קליסט הרגיל שלך, עצור וחפש את הייחודי-לכאן"
3. **הרחבת חופש ב-schema** — חבילה של 2-10 פריטים, לא 5-8 קבועים
4. **טמפ' 0.7+** ליצירתיות
5. **דוגמה מנוגדת** — "A's unknowns ≠ B's unknowns"

## תזכורת לעתיד

לפני בניית כל סוכן AI חדש בפרויקט, תשאל:
- [ ] האם נתתי לו רשימה מפורשת של נושאים? (אם כן, הסר אותה)
- [ ] האם ה-schema קובע מספר ידני של פריטים? (אם כן, הרחב טווח)
- [ ] האם temperature < 0.5? (אם כן, רק אם דיוק חיוני יותר מיצירתיות)
- [ ] האם יש דוגמה אחת בלבד של פלט? (סכנה — המודל יחקה אותה)
- [ ] האם ה-mock fallback משכפל את התבנית הזו? (כן = משתמש יראה תבנית גם כש-AI נופל)
